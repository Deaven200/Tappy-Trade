================================================================================
                    TAPPY TRADE - GAME DESIGN DOCUMENT
================================================================================

Date: 2026-01-04
Platform: Mobile (Primary), PC, Browser
Genre: 2D Trading/Economy Idle Game
Engine: Unity (Recommended)

================================================================================
                              CORE CONCEPT
================================================================================

A mobile-first trading game where players harvest resources from wild land,
sell them in markets, and expand their empire by buying more plots.

Key Features:
- Tap to harvest wild resources (wood, berries, herbs)
- Real-time production and regrowth
- Player market + Government market (dual economy)
- Guild/trading company system
- Market manipulation possibilities
- Simple sprites with sounds and animations

================================================================================
                            CORE GAME LOOP
================================================================================

1. Start with 1 free overgrown plot of land
2. Tap trees, berry bushes, herbs to harvest
3. Resources go into inventory (1000 slots per plot)
4. Sell resources to market for money
5. Use money to buy more land plots (exponentially more expensive)
6. Empty plots regrow over 4-12 hours
7. Expand empire with more plots and production

================================================================================
                          LAND PLOT SYSTEM
================================================================================

Starting State:
- Plot #1: FREE (tutorial plot)
- Each plot: 1000 inventory slots
- Wild â†’ Harvested â†’ Empty â†’ Regrow (4-12 hours)

Plot Costs (Exponential scaling):
- Plot #1: FREE
- Plot #2: $5,000
- Plot #3: $15,000
- Plot #4: $35,000
- Scaling: Each plot ~2-3x more expensive

Plot Structure (NEW MENU-BASED SYSTEM):
- Each PLOT contains 3 SUB-PLOTS (displayed as boxes at top of screen)
- Sub-plots can be:
  * WILD LAND: Grows resources over time (forest, herbs, berries)
  * BUILDINGS: Placed by player for processing/production

Sub-Plot Interactions:
- TAP: Harvest resources from wild land
- TAP & HOLD: Open building placement menu
- NPC Workers: Can be hired to auto-harvest sub-plots

Building Placement:
- Buildings replace wild sub-plots
- Cost: Resources + Money
- Example: Sawmill = 50 Wood + 20 Stone + $500
- Buildings process inventory items automatically

================================================================================
                          INVENTORY SYSTEM
================================================================================

Capacity:
- Base: 1000 slots per owned plot
- 1 plot = 1000 slots
- 2 plots = 2000 slots
- Scales with land ownership

Stacking:
- Wood: Stack to 100
- Berries: Stack to 50
- Herbs: Stack to 50
- Prevents inventory spam

================================================================================
                          MARKET SYSTEMS
================================================================================

--- GOVERNMENT MARKET (NPC/Automated) ---

Features:
- Unlimited supply/demand (always available)
- Dynamic pricing based on global player production
- Price formula: Base Price Ã— (Demand / Supply)
- Updates every 15-30 minutes

Example Prices:
Item      | Sell Price | Buy Price
----------|------------|----------
Wood      | $5         | $8
Berries   | $8         | $12
Herbs     | $10        | $15

Price Mechanics:
- Oversupply = prices drop
- Undersupply = prices rise
- Daily events can boost certain items

--- PLAYER MARKET (P2P Trading) ---

Features:
- Players post buy/sell orders
- Order book system (like stock exchange)
- Orders active for 24-48 hours
- 5% transaction fee
- Guild chat for negotiation

Market Manipulation:
- Guilds can coordinate buys to spike prices
- Counter: Purchase limits or government interventions

Production Automation:

- Worker assignments (offline progression)
